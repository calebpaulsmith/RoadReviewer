Michigan — road functional class & route designation

Organization: Michigan Department of Transportation (MDOT) — public ArcGIS Server, mdotgis.state.mi.us

Layer 353 — "Functional System" (the class code)

https://mdotgis.state.mi.us/arcgis/rest/services/Widget/NextGenPrFinderPub/FeatureServer/353

Polyline layer keyed on Michigan's LRS (PR + mileposts). Fields read: FunctionalSystem (smallint), PR. The FunctionalSystem coded-value domain (LrseFunctionalSystem):

codemeaning
0Non-Certified Roadway
1Interstate
2Other Freeway
3Other Principal Arterial
4Minor Arterial
5Major Collector
6Minor Collector
7Local
Quirk — retired segments. The layer is MDOT's editable production LRS; several overlapping segments at one point can be historical. Every query filters RHRetireDate IS NULL, or you read retired classifications.
Quirk — no urban/rural in the code. Unlike e.g. Tennessee's layer, the class code is bare FHWA 1–7. Urban vs Rural comes exclusively from the nationwide ACUB check below.
Quirk — legend. The map figure/legend colors are MDOT's own published drawingInfo.renderer for this layer (e.g. Minor Collector = yellow, Local = dark grey), read live from the layer's REST metadata — not colors invented by this tool.

Layer 543 — "Route System" (trunkline route designation)

https://mdotgis.state.mi.us/arcgis/rest/services/Widget/NextGenPrFinderPub/FeatureServer/543

Fields read: RouteDesignation, RouteNumber (same RHRetireDate IS NULL filter).

Quirk — trunkline-only names. Only designated Interstate / US / M routes are populated (e.g. "I-94 BL"). Local streets return nothing here — that's expected, and the Census TIGER layer below fills the street-name gap.

Indiana — road functional class & road name

Organization: Indiana Department of Transportation (INDOT) via the IndianaMap GIS platform, gisdata.in.gov

Layer 22 — "LRSE_Functional_Class" (the class code)

https://gisdata.in.gov/server/rest/services/Hosted/LRSE_Functional_Class/FeatureServer/22

Fields read: functional_class (smallint, domain dFunctionalClass — same FHWA 1–7 numbering as Michigan, no urban/rural embedded).

Quirk — record status. Indiana's analog to Michigan's retire-date filter is a status domain (dRecordCode: 0 Work In Progress … 5 Active … 7 Retired). Every query filters record_status=5 (Active).
Quirk — no name field at all. The functional-class layer carries no road name — not even a trunkline-only one — and is not LRS-keyed to any layer that does. Names come from a fully separate point query (below), so the Route column is blank more often for Indiana.
Quirk — legend. INDOT publishes this layer with a single-symbol renderer (every segment the same green, confirmed live 2026-07-05), which can't distinguish classes by color. In map figures and legends this tool therefore substitutes the standard FHWA class palette (byte-identical to the colors MDOT publishes for the same classes) — the substitution is disclosed on the figure itself. The class values are always INDOT's own.

Layer 15 — "Road_Centerlines_of_Indiana_2021" (road name)

https://gisdata.in.gov/server/rest/services/Hosted/Road_Centerlines_of_Indiana_2021/FeatureServer/15

Field read: st_full (e.g. "MERIDIAN ST"). The layer's own roadclass field returned null at every sample point and is not used.

Wisconsin — road functional class & road name (two layers, in sequence)

Organization: Wisconsin Department of Transportation (WisDOT) — ArcGIS Online hosted services (services5.arcgis.com, account wisnipsvki_WisDOT), published for damage-assessment use

Layer 3 — "Functional Class - State Trunk Network" (tried first)

https://services5.arcgis.com/0pgGLzT0Nh7FVjon/arcgis/rest/services/FFCL_gdb/FeatureServer/3

Covers interstates and state highways. Fields read: FED_FC_CD (string "1"–"7", bare FHWA code), HWYTYPE/HWYNUM/HWYDIR (route name, e.g. "USH 18 W"). Point-in-time snapshot — no retired-segment filter needed. If any segment answers here, the local-roads layer is not queried.

Quirk — two class fields. This layer carries both FED_FC_CD (the federal FHWA code — what the classifier uses) and FC_CD (WisDOT's own internal code — what the layer's published renderer/legend keys off). The map figures fetch both: FED_FC_CD for the verdict, FC_CD only for matching WisDOT's own colors.
Quirk — URB_TYPE unused. The layer has its own Urban/Rural field, but this tool deliberately ignores it: the nationwide ACUB polygon check is the single urban/rural source of truth for every state, so all states are judged by the same boundary dataset.

Layer 1 — "WI_Local_Roads_Flood_Damage_Assessment_Snapshot" (fallback)

https://services5.arcgis.com/0pgGLzT0Nh7FVjon/arcgis/rest/services/WI_Local_Roads_Flood_Damage_Assessment_Snapshot/FeatureServer/1

Covers county/municipal/town roads. Fields read: FNCT_CLS_CTGY_TYCD (WisDOT category code), ST_LABL_NM (street name — good local-street coverage).

Quirk — urban/rural embedded in the code. Unlike every other wired layer, this one encodes urban/rural into the class value itself (10 Rural Principal Arterial, 60 Urban Principal Arterial, 20/86 Minor Arterial, 30/96 Collector, 40 Rural Minor Collector, 45/97 Local). The tool strips that back to a bare FHWA 1–7 code before the shared federal-aid rule runs.
Quirk — code 96 ("Urban Collector Other") doesn't split major vs minor collector. It is mapped to Major Collector (5). This can never change a verdict: the major/minor split only matters for rural collectors, and every urban collector is federal-aid regardless.
Quirk — no formal legend domain. The category-code meanings aren't published as a coded-value domain; they were read from the layer's renderer uniqueValueInfos (its map symbology), which is also what the map figures use for colors.

All states — Urban vs Rural (ACUB)

Organization: USDOT Bureau of Transportation Statistics — National Transportation Atlas Database (NTAD), "2020 Adjusted Urban Area Boundaries", ArcGIS Online hosted (re-hosted via UDOT's UPLAN open-data portal)

Layer 0 — "Adjusted Urban Areas"

https://services.arcgis.com/xOi1kZaI0eWDREZv/arcgis/rest/services/NTAD_Adjusted_Urban_Areas/FeatureServer/0

Nationwide polygon layer of FHWA-adjusted 2020 Census urban boundaries. Fields read: NAME (e.g. "Kalamazoo, MI"), UACE (5-digit Census urban-area code), state_1. A point inside any polygon ⇒ Urban; otherwise Rural. This single layer answers urban/rural for every state, wired or not.

Quirk — boundary buffer floor. The point-in-polygon test retries with a buffer of at least 200 ft when the exact point misses, so a site a few feet on the wrong side of a road that itself touches the urban boundary still resolves Urban (a real field-observed failure mode).
Quirk — generalized geometry in figures. Boundary geometry drawn on the map/PDF is fetched with ~11 m simplification (maxAllowableOffset) to keep the payload small; the urban/rural determination itself always uses the full-precision service-side intersection.

All states — street names

Organization: U.S. Census Bureau — TIGERweb

Layer 8 — "Local Roads" (Transportation MapServer)

https://tigerweb.geo.census.gov/arcgis/rest/services/TIGERweb/Transportation/MapServer/8

Field read: NAME, pipe-joined when several streets fall in the search buffer (intersections). Backfills the street name wherever the state layer only knows designated routes (Michigan, and often Indiana). Failures here are non-fatal — classification still lands without a street name.

FIRMettes (flood map extracts)

Organization: FEMA — Map Service Center, Print FIRMette geoprocessing service
https://msc.fema.gov/arcgis/rest/services/NFHL_Print/MSCPrintB/GPServer/PrintFIRMette

Used by the "Download FIRMettes (ZIP)" button: for each site the browser submits a print job (submitJob?input_lat=…&input_lon=…&Print_Type=FIRMETTE&graphic=PDF), polls the job status, then downloads the produced PDF — the same flow FEMA's own Map Service Center portal uses. Each PDF is generated fresh by FEMA (~1 MB, typically seconds to a couple of minutes per site), which is why the batch is capped.

Basemap (visual reference only)

Organization: Esri — World Street Map (default), World Imagery + Boundaries & Places (switcher option)

The interactive map's background tiles come from Esri and play no role in any classification. The PDF report figures also composite Esri World Street Map tiles under the authoritative data layers (fetched with crossOrigin="anonymous" against Esri's CORS-enabled tile service, so a non-CORS response fails outright instead of silently tainting the canvas; the fetch is disclosed in the network log and credited on the figure). Viewing the map — or generating a report — necessarily reveals the viewed area to the tile provider; this is the one disclosure the client-side design can't remove (disclosed in the main page footer). The data layers in every figure remain pure vector drawings queried from the authoritative services above.

MN / IL / OH

No road-functional-class layer is wired for Minnesota, Illinois, or Ohio yet — sites there get the ACUB urban/rural check only, and the verdict column says so rather than guessing. The nationwide ACUB and TIGER layers above already cover them; each state's DOT publishes its NFC layer differently and needs the same per-state schema research MI/IN/WI received.

All layer names, field names, domains, and quirks on this page were confirmed against each service's live REST metadata (?f=json) — sources last verified 2026-07-01 through 2026-07-03. This tool classifies the road, never the project.
